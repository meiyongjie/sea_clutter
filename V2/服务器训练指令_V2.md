# 服务器训练指令：V2 海杂波距离维预测

## V2 核心改动（相对 V1）

| 改动 | V1 | V2 | 原因 |
|------|----|----|------|
| LSTM 架构 | fc(last_hidden)→8点并行 | Encoder-Decoder 自回归+Teacher Forcing | 并行输出无时序结构→直线 |
| GRU 解码器 | 输入全零 | 种子投影+Cross-Attention+残差跳连 | 全零无起点→输出均值 |
| PatchTST 配置 | seq=64, patch=8, stride=4, d=64 | seq=128, patch=4, stride=2, d=128, 3层 | 16个patch注意力退化 |
| 归一化 | 全局 MinMax | 分位数裁剪(1%~99%) MinMax | 尖峰压缩动态范围 |
| 损失函数 | SmoothL1+grad+std+dynamic | MSE+可选grad(0.2) | dynamic加权关注不可预测尖峰 |
| rollout_steps | 4（闭环训练） | 1（单步训练） | 闭环训练太早→模型学保守策略 |
| 早停 | 无 | patience=15 | 防过拟合 |

## 6. 重点判断标准

### ✅ 成功标准

```
1. 所有模型 test_mae_db < 4.36 (优于 Mean baseline)
2. 所有模型 test_std_ratio > 0.05 (非直线)
3. PatchTST std_ratio > 0.3 (有显著起伏)
4. 训练 loss 稳定下降，无震荡
5. 早停在合理 epoch 触发（30~60 之间）
```

### ⚠️ 需要关注

```
1. 某模型 std_ratio < 0.02 → 仍然是直线，需进一步调参
2. 训练 loss 不降 → 检查学习率，尝试 lr=0.01 (RNN) 或 lr=0.001 (PatchTST)
3. val_mae_db 震荡不降 → 降低 lr 或增大 batch_size
4. Rollout-4 std_ratio 急剧下降 → 正常的分布漂移，后续用 Scheduled Sampling 缓解
```

### ❌ 不要做的事

```
1. 不要在第一轮训练中改 seq_len / pred_len / split_seed
2. 不要同时改多个超参
3. 不要在单步预测未达标时就开 rollout_steps>1 训练
4. 不要加回 dynamic_loss_weight（V1 已证明它让模型关注不可预测的尖峰）
```

## 7. 显存不足处理

优先降低 batch_size：

```python
# 在对应 config 文件中修改

self.batch_size = 32   # 第二步
```

如果 PatchTST 仍然不足，可降低 d_model：

```python
self.d_model = 64      # 从 128 降到 64
self.d_ff = 128        # 从 256 降到 128
```

## 9. 训练后下一步

单步预测达标后（mae < 4.0, std_ratio > 0.3），可进入以下优化：

1. **Scheduled Sampling**：teacher_force_ratio 从 1.0 线性衰减到 0.0
2. **rollout_steps=2 训练**：渐进式闭环训练
3. **噪声注入**：训练时给解码器输入加 N(0, 0.01) 噪声
4. **grad_loss_weight 调参**：尝试 0.3 / 0.5 / 0.0 消融
5. **复数 IQ 模型**：在 PatchTST V2 基础上加 enc_in=2 的复数版本

但**必须等单步预测先达标**，否则任何优化都建立在沙上。
